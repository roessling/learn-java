Konfiguration

Wenn Sie das Programm zum ersten Mal starten, müssen Sie zuerst die Konfiguration vornehmen. Zu Beginn sieht das Fenster wie folgt aus:

In das obere Feld muss der Pfad zum JDK-Ordner eingetragen werden. Unten muss der Pfad zur JUnit-Datei stehen, die typischerweise im selben Ordner wie das Programm vorhanden ist. Beide Felder können mithilfe der Knöpfe "Durchsuchen..." ausgefüllt werden. Sollte dieses Fenster unklar sein, lesen Sie den Artikel über die Konfiguration.

Bearbeitung von Aufgaben

Sobald die Konfiguration abgeschlossen ist, kann die Bearbeitung der Aufgaben schon beginnen! Das Hauptfenster des Programms sehen Sie auf dem unten stehenden Bild. Alle Elemente sind markiert und werden im Folgenden genauer beschrieben.

Alle bereits geladenen Aufgaben werden in einer Liste auf der linken Seite angezeigt (1). Sollten noch keine Aufgaben vorhanden sein, lesen Sie den Abschnitt Importieren von Aufgaben. Um eine Aufgabe zu bearbeiten, klicken Sie sie an. Sobald eine Aufgabe ausgewählt wurde, werden alle Knöpfe aktiviert.

Nachdem eine Aufgabe ausgewählt wurde, erscheint auf der rechten Seite eine Aufgabenstellung (2) zu dieser Aufgabe. Lesen Sie die Aufgabenstellung sorgfältig, um die Aufgabe zu lösen.

Jetzt können Sie in der Mitte des Fensters (3) Ihren eigenen Code schreiben, um die ausgewählte Aufgabe zu lösen. Geschriebener Code wird für die Dauer einer Sitzung gespeichert, bis Sie das Programm schließen. Um beim nächsten Start des Programms am gleichen Code weiterarbeiten zu können, müssen Sie diesen mit dem "Speichern"-Knopf (4) hinterlegen.

Wenn Sie mit Ihrem Code fertig sind oder einen Kompilierversuch starten möchten, betätigen Sie den "Ausführen"-Knopf (5). Im unteren Teil des Fensters (6) werden die Ergebnisse der Ausführung Ihres Codes angezeigt. Wird Ihre Code fehlerfrei kompiliert und kann ausgeführt werden, werden die Ergebnisse der Tests im Reiter "Tests" angezeigt. Sollten nicht alle Tests das richtige Ergebnis erzielen, werden bei öffentlichen Tests der Aufruf und das erwartete Ergebnis angezeigt. Kommt es bei der Ausführung zu einem Fehler (Exception bzw. Error), erscheint der Typ des Fehlers und die Zeile, in der er auftritt. Falls der geschriebene Code nicht fehlerfrei kompiliert werden kann, werden die Meldungen des Compilers im Reiter "Compiler" angezeigt. Lassen Sie sich von einer ganzen Reihe von Fehlermeldungen nicht entmutigen - manchmal verursacht ein kleiner Fehler gleich mehrere. Sollten Sie anhand dieser Meldungen nicht feststellen können, was Sie an Ihrem Code ändern müssen, lohnt sich oft ein Blick in den Reiter "Precompiler". Hier werden Fehler erklärt, die der Compiler unverständlich formuliert oder gar nicht erst erkennt.

Sollten Sie eine Aufgabe mehrfach lösen wollen, lässt sich der gespeicherte Code mit dem Knopf "Zurücksetzen" (7) einfach löschen. Dann können Sie mit der Bearbeitung von vorne beginnen.

Importieren von Aufgaben

Sobald neue Aufgaben als Download zur Verfügung stehen, können Sie diese einfach herunterladen und in das Programm einlesen. Verwenden Sie dafür den Knopf "Importieren" (8). Es öffnet sich ein Dialog, in dem Sie die heruntergeladene Datei auswählen können. Die darin enthaltenen Aufgaben werden entpackt und gespeichert, sodass sie ab sofort bei jedem Start des Programms zur Verfügung stehen.

Weitere Funktionen

Mit dem Knopf "Konfiguration" (9) lässt sich jederzeit die Konfiguration verändern. Beachten Sie, dass eine gültige Konfiguration nötig ist, um das Programm zu verwenden.

Der Knopf "Wissensdatenbank" (10) öffnet eine Datenbank mit einer Sammlung von Artikeln über Konzepte der Programmiersprache Java. Sollten Sie einmal nicht wissen, wie man bestimmte Konstrukte der Sprache anwendet, finden Sie hier Erklärungen und Beispiele, die Ihnen weiterhelfen.

Mit dem Knopf "Hile" (11) öffnen Sie diese Hilfe vom Programm aus.
